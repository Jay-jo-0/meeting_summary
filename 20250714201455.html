TF 테스트 회의록

회의 핵심 요약

이번 회의는 AI 기술(특히 ChatGPT)을 활용한 업무 자동화 및 생산성 향상 방안에 대해 심도 깊게 논의했습니다. 특히 회의록 자동 요약, 리서치 자료 분석, 뉴스 번역 및 요약, 리포트 제작 자동화, 증시 시황 생성 등 다양한 실제 적용 사례를 공유하며 AI의 잠재력을 확인했습니다.

주요 논의 내용은 다음과 같습니다:

  • 회의록 자동화: 클로바노트와 ChatGPT를 연계하여 전체 회의 텍스트 기반의 상세 요약 및 마인드맵 생성.
  • 정보 분석 자동화: 딥리서치 기능 활용, 대량 뉴스 자동 번역 및 요약 (API 연동), 시황 코멘트 자동 생성.
  • 리포트 모바일 최적화: PDF 리포트를 HTML 형식으로 변환하여 모바일 가독성 향상 및 AI 해설 추가.
  • AI 활용의 한계 및 극복 방안: 토큰 제한, 비용, 속도, 정확도 문제 인식 및 RAG(Retrieval-Augmented Generation) 도입을 통한 개선 계획.
  • TF 활동 지원: 텔레그램 봇을 활용한 회의록 자동 생성 및 구글 드라이브 저장 기능 개발.

결론적으로, AI는 단순한 자동화를 넘어 업무 생산성을 혁신적으로 높이는 강력한 도구이며, 향후 RAG 시스템 구축을 통해 더욱 고도화된 정보 활용이 가능할 것으로 기대됩니다.

상세 회의 내용

1. 회의 요약 자동화 전략

  • 클로바노트와 ChatGPT 연계:
    • 클로바노트가 제공하는 회의 텍스트를 ChatGPT의 프롬프트로 활용하여 원하는 형식으로 요약.
    • 클로바노트 자체 요약은 너무 간략하여(세 줄 요약 수준) 전체 텍스트를 활용하는 방식이 더 효과적임을 확인.
    • 과거 회의에서도 이 방식을 통해 전체 텍스트 기반의 상세 요약을 진행했음.
  • 마인드맵 생성 자동화:
    • 인터넷 마인드맵 툴을 활용하여 마크다운 형식의 텍스트를 마인드맵으로 변환.
    • ChatGPT에게 마크다운 형식으로 요약 및 정리해달라고 요청하여 마인드맵 생성을 자동화할 수 있음.
    • 회의 내용(예: UX 노하우 관련)을 전체 텍스트와 함께 ChatGPT에 넣어 관련 내용을 문서로 정리한 후 마인드맵에 연결하여 즉시 조회 가능하도록 구현.
    • 이를 통해 복잡한 회의 내용을 시각적으로 구조화하고, 특정 주제에 대한 상세 내용을 빠르게 찾아볼 수 있는 시스템을 구축했음.

2. 정보 리서치 및 분석 자동화

  • ChatGPT의 딥리서치 기능 활용:
    • ChatGPT의 딥리서치 기능은 여러 웹사이트를 자동으로 탐색하고 결과를 종합하여 제공하므로, 자료 조사에 매우 유용함.
    • 특히 생소한 산업 분야를 리서치할 때, 기초 검색부터 시작하는 시간을 크게 단축시키고, 출처까지 제공하여 정보의 신뢰도를 높일 수 있음.
    • 직접 손으로 정보를 수집하는 것보다 훨씬 빠르고 효율적인 리서치가 가능함. (울트라 버전 사용)
  • 대량 뉴스 자동 번역 및 요약 (API 활용):
    • 하루에 약 5,000건의 이메일 뉴스를 수신하는 상황에서, 이를 일일이 확인하고 번역하는 것은 불가능.
    • 메일 수신 즉시 GPT API에 연동하여 한글로 번역하고 핵심 내용을 요약하도록 자동화.
    • 긴 영어 기사를 짧은 한글 요약본으로 변환하여 텔레그램으로 즉시 전송받는 시스템 구축.
    • 이를 통해 이동 중이나 짧은 시간에도 주요 뉴스를 빠르게 파악하고 대응할 수 있게 됨.
    • 데이터 출처: 런던 스톡 익스체인지 그룹(LSEG), 블룸버그 등 전문 금융 데이터 그룹.
  • 시황 코멘트 자동 생성:
    • 수집된 뉴스 데이터와 자체 보유한 가격 변동 데이터를 ChatGPT에 입력하여 원/달러 및 달러 시황 코멘트를 자동 생성.
    • 단순 프롬프트 질문보다 상세한 데이터(예: 어제/오늘 가격 차이, 관련 뉴스)를 제공함으로써 훨씬 구체적이고 심층적인 결과물 도출.
    • 예시: 멕시코 관세 부과, 트럼프 관련 이슈 등 구체적인 내용을 포함한 시황 분석.
    • 기존에 수작업으로 30분 이상 소요되던 일일 시황 작성을 AI를 통해 몇 분 내로 단축하여 생산성 극대화.

3. 리포트 제작 자동화 및 모바일 최적화

  • PDF 리포트 HTML 변환 및 AI 해설 추가:
    • 기존 PDF 리서치 리포트는 모바일 기기에서 가독성이 매우 떨어짐 (확대/축소 및 스크롤 불편).
    • AI를 활용하여 PDF 내용을 모바일 친화적인 HTML 형식으로 자동 변환.
    • 리포트 발행 시, AI가 자동으로 표지를 생성하고 간단한 요약을 제공.
    • 특히 제약 바이오, 건설 등 전문 용어가 많은 어려운 리포트에 대해 AI가 요약 및 해설을 추가하여 내용 이해도를 높임.
    • 애널리스트의 코멘트까지 AI가 해설하여 실시간으로 제공함으로써 자료의 파급력을 증대.
  • 증시 요약 자동 생성:
    • 뉴스, 기업 데이터(LSEG, 블룸버그), 기업 주가 및 재무 데이터를 종합하여 장 마감 후 증시 요약 자동 생성.
    • 시황 코멘트와 가격 변동 데이터를 AI에 입력하여 생성.
    • 기존에는 뉴스 확인, 가격 데이터 분석 등 30분 이상 소요되던 작업을 AI가 자동화하여 시간을 대폭 절약.
    • 생성된 내용을 최종 확인 및 수정 후 바로 배포 가능.

4. AI 활용 시 한계점 및 극복 방안

  • 토큰 제한 문제:
    • 대규모 데이터를 처리할 때 API의 토큰 제한(예: 100만~200만 토큰)이 큰 제약으로 작용.
    • 하루 5,000건의 뉴스처럼 방대한 데이터는 몇 시간 만에 토큰 제한을 초과함.
    • 청킹(Chunking) 방식 활용: 데이터를 여러 작은 덩어리(예: 10만 토큰씩)로 쪼개어 순차적으로 처리한 후, 그 결과들을 다시 합쳐 최종 결과를 도출.
  • 청킹 방식의 문제점:
    • 높은 비용: API 호출 횟수가 증가하여 비용이 많이 발생.
    • 느린 속도: 여러 번의 처리 단계를 거치므로 전체 처리 시간이 길어짐.
    • 정확도 저하: 여러 단계를 거치면서 자료 변형 가능성이 있어 정확도가 떨어질 수 있음.
  • RAG (Retrieval-Augmented Generation) 도입 계획:
    • 현재 직면한 토큰 제한 및 정확도 문제를 해결하기 위한 차세대 방안으로 RAG 시스템 구축을 준비 중.
    • RAG 원리: 사용자의 질문을 검색 모델이 벡터 DB에서 관련 외부 정보(뉴스, 문서 등)를 검색하여 GPT/Gemini와 같은 LLM에 제공, 이를 통해 LLM이 더 정확하고 풍부한 답변을 생성.
    • 기존 SQL 검색의 한계: 키워드 기반 SQL 검색은 정확도가 떨어지고 불필요한 데이터가 많이 추출됨 (예: '트럼프' 관련 뉴스 검색 시 방대한 결과).
    • 벡터화 검색의 장점: 모든 뉴스 데이터를 벡터화하고, 질문도 벡터로 변환하여 DB에 던지면 의미론적 유사성을 기반으로 훨씬 빠르고 정확한 결과를 얻을 수 있음.
    • RAG 구축을 통해 무거운 데이터 처리 작업을 가볍고 효율적으로 수행할 수 있을 것으로 기대.
    • 현재 개인적으로 RAG 시스템을 구축하며 API 활용의 한계를 극복하려 노력 중.

5. TF 활동 지원을 위한 AI 도구 개발

  • 텔레그램 봇 기반 회의록 자동 생성기:
    • TF 활동을 지원하기 위해 텔레그램 봇을 개발.
    • 사용자가 텍스트 파일을 봇에 입력하면, 자동으로 회의록을 생성하여 반환하고 구글 드라이브에 저장.
    • 과거 TF 회의록 작성 경험을 바탕으로 수동 작업을 자동화하기 위해 개발.
    • 첫 회의록 생성 후, 추가 텍스트 입력 시 기존 회의록과 합쳐 전체 회의록을 업데이트하는 기능 포함.
    • 회의 인사이트, 향후 계획 등 특정 섹션을 프롬프트에 포함하여 자동으로 생성되도록 설정 가능.
    • 현재는 TXT 파일만 지원하지만, 오디오 파일이나 PDF도 텍스트로 변환하여 입력할 수 있도록 확장 가능.
    • (회의 당시) 일시적인 오류로 작동하지 않았으나, 곧 수정하여 공유 예정.
  • 개발 환경 및 도구:
    • 주로 파이썬을 활용하여 개발.
    • 백엔드부터 프론트엔드(텔레그램 봇)까지 모두 직접 구현했으나, 이 과정에서 ChatGPT(LLM)의 도움을 매우 크게 받음.
    • 과거 구글링을 통한 자동화보다 훨씬 복잡하고 광범위한 개발이 가능해졌음을 강조.
    • 텔레그램은 업무용이 아닌 개인적인 목적으로 활용 사례를 보여주기 위함이며, 내부 앱 개발을 통해 회사 시스템에 통합 가능함을 언급.

6. 기타 AI 활용 아이디어 및 인사이트

  • 실시간 회의록 및 지식 검색 시스템 (티로):
    • '티로'와 같은 실시간 텍스트 변환 및 요약 도구를 도입하면, 회의 중에도 실시간으로 회의록을 생성하고 요약 가능.
    • 만약 전사 회의록이 모두 벡터화된 DB에 저장되어 있다면, 회의 중 특정 주제에 대해 관련된 과거 회의 내용이나 다른 부서의 문서를 즉시 검색하여 보여줄 수 있음.
    • 이는 회의의 효율성과 의사결정의 질을 혁신적으로 높일 수 있는 방안.
  • AI를 통한 생산성 향상 경험:
    • 올해(회의 시점 기준) AI 활용 업무를 시작했으며, 불과 몇 개월 만에 이 모든 시스템을 구축했음을 강조.
    • GPT와 같은 LLM의 도움 없이는 불가능했을 것이며, 개인의 업무 생산성을 비약적으로 높이는 데 결정적인 역할을 함.

회의 인사이트

이번 회의를 통해 AI, 특히 LLM(대규모 언어 모델)이 단순한 보조 도구를 넘어 업무 프로세스와 생산성을 혁신적으로 변화시킬 수 있는 핵심 동력임을 명확히 확인했습니다. 발표자가 공유한 다양한 실제 적용 사례들은 AI가 특정 부서나 전문가 집단에만 국한되지 않고, 개인의 업무 효율성까지 극대화할 수 있음을 보여주었습니다.

특히 주목할 만한 인사이트는 다음과 같습니다:

  • AI 활용의 접근성: 전문 개발 지식이 없더라도 LLM의 도움을 받아 복잡한 자동화 시스템을 구축할 수 있다는 점은 AI 기술 도입의 문턱을 낮추는 중요한 시사점을 제공합니다. 이는 전사적인 AI 활용 문화 확산에 긍정적인 영향을 미칠 것입니다.
  • 데이터 활용의 중요성: AI의 성능은 입력되는 데이터의 질과 양에 비례한다는 점이 다시 한번 강조되었습니다. 단순한 프롬프트 입력보다는, 기업 내부의 정제된 데이터(뉴스, 시황, 재무 데이터 등)를 AI와 결합할 때 훨씬 심층적이고 유의미한 결과물을 도출할 수 있습니다.
  • 한계 극복을 위한 지속적인 연구: 토큰 제한, 비용, 속도, 정확도와 같은 AI 활용의 현실적인 한계점을 인지하고, 이를 극복하기 위한 RAG와 같은 차세대 기술 도입을 모색하는 자세는 매우 중요합니다. 이는 AI 기술의 지속적인 발전을 위한 필수적인 과정입니다.
  • 협업 및 지식 공유의 가치: AI를 통해 자동화된 정보(회의록, 리포트, 시황 등)를 실시간으로 공유하고 활용하는 시스템은 부서 간, 개인 간 지식 격차를 줄이고 시너지를 창출하는 데 기여할 수 있습니다. 특히 전사적인 지식 DB 구축은 미래 경쟁력의 핵심이 될 것입니다.

향후 계획 및 방향성

  1. RAG(Retrieval-Augmented Generation) 시스템 구축 가속화:
    • 현재 직면한 AI 활용의 기술적 한계(토큰 제한, 속도, 비용)를 극복하기 위해 벡터 DB 기반의 RAG 시스템 구축에 집중.
    • 이를 통해 대량의 내부/외부 데이터를 효율적으로 검색하고, 더욱 정확하고 심층적인 AI 답변을 생성하는 것을 목표로 함.
    • 특히 의미 기반의 벡터 검색을 통해 복잡한 질문에도 최적화된 정보를 제공할 수 있도록 고도화.
  2. 회의록 자동화 봇 안정화 및 전사 확산 검토:
    • 현재 개발된 텔레그램 기반 회의록 자동 생성 봇의 안정성을 확보하고, TF 내부에서 적극적으로 활용하여 피드백 수집.
    • 장기적으로는 텔레그램을 대체할 내부 업무용 앱 개발을 통해 회사 시스템에 통합하는 방안을 검토하여 보안 및 접근성을 강화.
    • 오디오 파일 및 PDF 파일 입력 기능 추가 등 사용자 편의성 증대를 위한 기능 개선 추진.
  3. AI 기반 지식 관리 시스템 구축 검토:
    • '티로' 사례와 같이 모든 전사 회의록 및 중요 문서를 벡터화하여 DB에 저장하고, 실시간으로 검색 및 요약할 수 있는 지식 관리 시스템 도입을 검토.
    • 이를 통해 회의 중 필요한 정보를 즉시 조회하고, 부서 간 지식 공유를 활성화하여 의사결정의 질을 높이는 데 기여.
  4. AI 활용 교육 및 문화 확산:
    • 발표자의 경험처럼, 비개발 직군도 LLM의 도움을 받아 업무 자동화를 구현할 수 있음을 전파.
    • AI 활용 교육 프로그램을 기획하여 전 직원의 AI 리터러시를 향상시키고, 개개인의 업무 생산성 향상을 위한 AI 도구 활용을 장려.
    • 다양한 부서의 업무에 AI를 적용할 수 있는 아이디어를 발굴하고 시범 적용하는 프로젝트 추진.
  5. 지속적인 AI 기술 트렌드 모니터링 및 적용:
    • 빠르게 변화하는 AI 기술 트렌드를 지속적으로 모니터링하고, 업무에 적용 가능한 새로운 기술이나 모델을 적극적으로 도입.
    • 비용 효율성, 속도, 정확도 등 다양한 측면을 고려하여 최적의 AI 솔루션을 선택하고 활용.
